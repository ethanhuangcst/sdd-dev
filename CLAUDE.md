# CLAUDE.md - sdd-dev Project Instructions

## Process Assets

Before starting any task, read `docs/adr/README.md` for a summary of all architectural and process decisions.

Only read individual ADR files when they are directly relevant to the current task.

Do not contradict any adopted ADR without first creating a superseding ADR.

## Workflows

- New feature development → use `/new-requirement` skill
- Requirement change → use `/change-requirement` skill
- Post-task retrospective → use `/retrospective` skill
- DoD validation → use `/dod` skill
